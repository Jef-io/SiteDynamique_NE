Ecosystème

Architecture

Pour ce prjet, nous avons décidé d'utiliser l'architecture suivante :

  • Le projet utilise Parcel en tant que 'Module Bundler'. Cet outil va permettre de gérer avec aise le différentes dépendances ou librairies d'installées. Aussi, parcel permet de générer une version pour le déploiement qui sera plus adaptée pour les navigateurs que le code que l'on créé. Nous pouvons donc moins nous soucier de la manière dont nous avons conçu le projet. Parcel permet aussi d'utiliser des préprocesseurs css comme sass, qui est plus agréable pour le développeur. De plus, parcel n'a besoin de 0 configuration, on peut directement travailler avec sans passer par 10h de mise en place.

  • Dans le dossier racine, il y a tous les fichiers utiles pour le développement (gitignore, package.json, etc...).

  • Le dossier src (où se trouve le magnifique code) est composé de 4 sous dossiers :

    • Assets, qui comporte tous les médias, rangés par catégorie

    • js ou se trouvent tous les scripts JS utiles au bon fonctionnement du site

    • pages contient tous les fichiers html, eux mêmes rangés dans des dossiers de sous menu

    • scss contient les différents fichiers scss

  • Nous avons fait le choix d'utiliser le préprocesseur scss car il nous permet différentes fonctionnalités agréables à utiliser. Par exemple, nous pouvons stocker des couleurs dans des variables et accéder à une classe css à l'intérieur de la classe parent.

Ce choix d'architecture a l'avantage d'être simple à apréhender et à comprendre. Il est donc difficile de se perdre et une architecture propre incite à rédiger du code propre.
